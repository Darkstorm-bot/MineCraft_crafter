📋 Prerequisites

1 Required Software:
  • Python 3.10+ - For the backend API and orchestrator
  • Node.js 18+ - For the Minecraft bot (mineflayer)
  • Git - For version control
  • Ollama or vLLM - For running local LLMs
  • Minecraft Java Edition (via TLauncher or official launcher)
💡 Note: This system uses a dual-agent architecture with an Architect (planning) and Engineer (execution) working together to build structures in Minecraft autonomously.

🔧 Installation

Option A: Using start.bat (Windows - Recommended)

1 Run the automated installer: 
start.bat

This script will automatically:

  • Check and install Python packages
  • Create a virtual environment (.venv)
  • Install Node.js dependencies
  • Check if Ollama is running (start it if not)
  • Provide connection instructions for Minecraft

Option B: Manual Installation

1 Set up Python environment: 
cd minecraft_autonomous_builder
python -m venv .venv
.venv\Scripts\activate  # Windows
# or
source .venv/bin/activate  # Linux/Mac
pip install -r requirements.txt
2 Install Node.js dependencies: 
npm install
3 Initialize the database: 
python scripts/init_db.py
4 Copy environment file: 
cp .env.example .env

Edit .env with your configuration (see LLM Configuration section)

🎮 Connecting to Minecraft

Step 1: Start Your Minecraft World

1 Open Minecraft and create/load a world:
  • Game mode: Creative (recommended for building)
  • Cheats: Enabled
  • Difficulty: Peaceful (optional, prevents interruptions)
2 Open the world to LAN:
  • Press Esc in-game
  • Click "Open to LAN"
  • Select game mode (Creative recommended)
  • Click "Start LAN World"
  • Note the port number shown (e.g., "Local game hosted on port 25565")

Step 2: Configure Bot Connection

3 Update bot configuration:

The bot connects via mineflayer. Create or edit bot/config.json:

{
  "host": "localhost",
  "port": 25565,
  "username": "BuilderBot",
  "version": "1.20.4",
  "auth": "offline"
}

Step 3: Start the Bot

4 Run the Minecraft bot: 
npm run build
node dist/index.js

The bot will connect to your Minecraft world and wait for commands from the API.

⚠️ Important: Make sure your Minecraft username doesn't conflict with the bot's username. If you're playing as "Steve", name the bot "BuilderBot" or something different.

🚀 TLauncher Setup Guide

What is TLauncher?

TLauncher is a free Minecraft launcher that allows you to easily manage multiple Minecraft versions and modpacks. Here's how to set it up with the Autonomous Builder:

Step 1: Install TLauncher

1 Download TLauncher:
  • Visit tlauncher.org
  • Download the version for your operating system
  • Install and run TLauncher

Step 2: Select Minecraft Version

2 Choose compatible version:
  • In TLauncher, select version 1.20.4 (recommended)
  • Or any version listed in configs/minecraft_versions.yaml
  • Click "Enter the Game"

Step 3: Configure Multiplayer Settings

3 Enable multiplayer in TLauncher:
  • Go to Options → Multiplayer Settings
  • Ensure "Multiplayer Game" is enabled
  • Set "Visible to LAN" to ON

Step 4: Open World to LAN

4 Start your world:
  • Create a new world or load existing one
  • Set game mode to Creative
  • Enable Cheats
  • Once in-game, press Esc → "Open to LAN"
  • Click "Start LAN World"
  • Note the port (usually 25565)

Step 5: Connect the Bot

5 Configure bot for TLauncher:

TLauncher uses offline mode authentication. Update your bot config:

{
  "host": "localhost",
  "port": 25565,
  "username": "BuilderBot",
  "version": "1.20.4",
  "auth": "offline"
}

The "auth": "offline" setting is crucial for TLauncher compatibility!

✅ TLauncher Tip: TLauncher makes it easy to switch between Minecraft versions. If you encounter compatibility issues, try switching to version 1.20.4 which is well-tested with this builder system.

🤖 LLM Configuration

The system supports multiple LLM backends. Configure them in your .env file or through the GUI.

Available LLM Options

Provider Environment Variable Example Value Notes
Ollama (Local) OLLAMA_MODEL llama3.2, mistral, codellama Free, runs locally
Ollama Base URL OLLAMA_BASE_URL http://localhost:11434 Default Ollama endpoint
Architect Model ARCHITECT_MODEL ollama/llama3.2 For planning & design
Engineer Model ENGINEER_MODEL ollama/codellama For code generation
OpenAI OPENAI_API_KEY sk-... Paid, high quality
Anthropic ANTHROPIC_API_KEY sk-ant-... Paid, excellent reasoning
vLLM VLLM_BASE_URL http://localhost:8000/v1 Fast local inference

Configuring Ollama (Recommended for Local Use)

1 Install Ollama:
  • Visit ollama.ai
  • Download and install for your OS
  • Ollama will run as a background service
2 Pull required models: 
ollama pull llama3.2
ollama pull codellama
ollama pull mistral
3 Verify Ollama is running: 
ollama list
curl http://localhost:11434/api/tags

Configuring vLLM (High-Performance Local)

1 Install vLLM: 
pip install vllm
2 Start vLLM server: 
python -m vllm.entrypoints.api_server \
  --model meta-llama/Llama-2-7b-chat-hf \
  --host 0.0.0.0 \
  --port 8000

Example .env File

# LLM Configuration
ARCHITECT_MODEL=ollama/llama3.2
ENGINEER_MODEL=ollama/codellama
OLLAMA_MODEL=mistral
OLLAMA_BASE_URL=http://localhost:11434

# Optional: Cloud providers
# OPENAI_API_KEY=sk-your-key-here
# ANTHROPIC_API_KEY=sk-ant-your-key-here

# Application settings
APP_ENV=development
DATABASE_URL=sqlite:///data/mempalace.db

▶️ Running the System

Quick Start Commands

1 Start the API server: 
uvicorn src.api.app:app --reload --host 0.0.0.0 --port 8000

The API will be available at http://localhost:8000

2 Start the Minecraft bot: 
npm run build
node dist/index.js
3 Access the API documentation:

Open your browser to http://localhost:8000/docs for interactive API documentation (Swagger UI)

Using the Build Pipeline

POST /projects

Create a new building project

{
  "name": "My Castle",
  "description": "A medieval castle with towers and walls",

  "size": "large"

🔧 Troubleshooting

Common Issues

Bot Can't Connect to Minecraft

  • Ensure Minecraft world is open to LAN
  • Check the port number matches your config
  • Verify firewall isn't blocking the connection
  • Try using "auth": "offline" for TLauncher
  • Make sure bot username differs from your player username

Ollama Not Responding

  • Check if Ollama is running: ollama list
  • Restart Ollama service
  • Verify OLLAMA_BASE_URL in .env
  • Ensure models are pulled: ollama pull llama3.2

API Server Won't Start

  • Check if port 8000 is already in use
  • Verify all dependencies are installed: pip install -r requirements.txt
  • Check database file exists in data/ directory
  • Run python scripts/init_db.py to initialize

Build Quality Issues

  • Use more descriptive project descriptions
  • Try different LLM models for better results
  • Adjust temperature settings in configs/model_routing.yaml
  • Enable vision verification for better accuracy
