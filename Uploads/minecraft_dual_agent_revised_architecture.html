Minecraft autonomous builder — revised dual-agent architecture, gap analysis and MemPalace schema

Two root causes drive everything else. Fix these two and the remaining eight either resolve or become tractable. The // build confusion is listed third because it's a hard stop when writing actual integration code — but it doesn't generate the others.
Critical — root cause #1
Coordinate collision: no spatial index
The LLM hallucinates coordinates with full confidence. Nothing checks whether those blocks overlap existing structures before they're placed. Every module placement is a gamble unless a real 3D index is queried first.
Fix: coord_index table in SQLite. All module extents are registered before any paste. The accessor runs a bounding-box intersection query on every proposed block range. Reservation is atomic — crash between loads can't leave a partial lock.
Critical — root cause #2
Single-agent context window saturation
One 19–27b model holds the full project plan, memory state, current blueprint, and correction loop simultaneously. At 19–27b the window is finite — coherence collapses as the project grows. Every other quality problem downstream is amplified by this.
Fix: two agents, sequential not parallel. Agent A fully unloads before Agent B loads. State never lives in context — it lives in MemPalace (SQLite on disk). VRAM is never double-occupied.

High
MemPalace has no actual backend
"Wings and Halls" as a concept is fine but without a real storage layer it's just a metaphor the LLM will hallucinate into. No atomic writes, no spatial queries, no version history.
Needs: SQLite + Python accessor class. Agents never touch raw SQL.
High
No rollback or checkpoint system
If the bot crashes mid-batch or a module places incorrectly, there's no recovery path. Build log entries don't include enough state to resume from a checkpoint.
Needs: checkpoint_state JSON written to build_log after every batch.
High
Block ID version mismatch
Block IDs changed fundamentally between 1.12 and 1.13+. A static dictionary breaks across versions without a compatibility shim tied to mc_version.
Needs: mc_version field in projects. Agent B validates block IDs against this during its structured critique pass.
Medium
Vision critique is unstructured
Asking LLaVA "is the nosecone pointy?" is too vague for reliable corrections. Produces inconsistent results that can't gate a re-entry decision.
Needs: structured diff prompt — expected block counts, symmetry score, numeric 0–100 quality output.
Medium
No terrain or inventory pre-check
Pipeline assumes flat terrain and infinite materials. No step clears ground, loads chunks around the origin, or validates bot inventory against the material manifest.
Needs: preflight phase before any block is placed. Terrain scan + flatten, chunk loading, inventory diff against material_manifest.
Medium
No convergence gate
No stopping criterion defined. Without a delta score threshold or max-iteration cap, agents will loop indefinitely refining an already-good blueprint.
Needs: delta < 5% = auto-approve; cycle ≥ 3 = pick best from history; approval_flag = immediate proceed.
Medium
No scale calibration or unit grounding
The LLM has no reference for how "15×15×20 blocks" looks in-world. Without a block-to-metre table injected into the prompt, proportions will be consistently wrong.
Needs: block reference table in the Architect's system prompt. 1 block = 1 m³. Include known landmark dimensions for grounding.

Convergence gate — runs between every Agent B unload
A — delta low
delta_score < 5% — blueprint barely changed. Agent B found almost nothing worth altering. Auto-approve and proceed to execution.
B — max cycles hit
iteration_count ≥ 3. Pick the highest-scored blueprint version from MemPalace history. Prevents runaway loops on genuinely contested designs.
C — explicit approval
approval_flag = TRUE. Agent B found zero issues in the validation pass. Proceed immediately without any further load cycle.
Execution layer no LLM involved
Preflight
Terrain scan + flatten around origin. Chunk loading for build radius. Inventory diff against material_manifest. Coord reservation lock written to coord_index for the approved blueprint's full extent.
Fails fast if inventory is short or terrain scan finds an obstruction outside the cleared radius
Batch builder
Approved JSON → .schem file via mcschematic → WorldEdit FAWE paste per module. Checkpoint_state JSON written to build_log after each batch. On resume, build_log is read to skip completed batches.
WorldEdit FAWE handles paste. Baritone handles navigation between paste origins. These are separate systems and called separately.
Vision verify
Screenshot per build phase → LLaVA with structured diff prompt: expected block counts per module, bilateral symmetry score, 0–100 quality integer. If score ≥ 80%, mark phase complete.
score < 80% → correction context tagged, written to MemPalace as vision_critique
Re-entry path: If vision score < 80%, the correction context is written to MemPalace as vision_critique_vn and the agent loop restarts — but Agent A loads with the critique already in MemPalace. It refines only the flagged sections. It does not re-plan the entire project. The convergence gate's iteration_count continues from where it left off.
MemPalace is a SQLite database. Both agents interact exclusively through a Python accessor class — they never write raw SQL. The accessor handles spatial collision queries, version bumping, atomic commits, and reservation locking. The coord_index table is the structural core: it's what makes collision detection real rather than a prompt instruction the LLM can ignore.
